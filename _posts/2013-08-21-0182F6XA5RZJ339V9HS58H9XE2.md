# キーワード
- Apache
- Tomcat

# 関連
- [障害メモ/Tomcat関連]({% post_url 2013-08-21-0182F6V5T8S5X1AK2SPRGK2ZD9 %})

# 現象
AJPでTomcatに接続したいが、
ブラウザから確認できない。

## Apacheのエラーメッセージ
- /etc/httpd/logs/error_log

```
[Sun Oct 17 23:48:46 2010] [error] proxy: AJP: disabled connection for (localhost)
[Sun Oct 17 23:49:05 2010] [error] (111)Connection refused: proxy: AJP: attempt to connect to 127.0.0.1:8009 (localhost) failed
[Sun Oct 17 23:49:05 2010] [error] ap_proxy_connect_backend disabling worker for (localhost)
[Sun Oct 17 23:49:05 2010] [error] proxy: AJP: failed to make connection to backend: localhost
```
どうもローカルに接続できないとかって。

## telnetのエラーメッセージ
8080番へtelnetしてみる。やっぱり繋がってない。
```
$ telnet localhost 8080
Trying 127.0.0.1...
telnet: connect to address 127.0.0.1: Connection refused
```
そもそも8080が上がっていないのでは。

## リスンしているポートを確認
```
$ netstat -anp
```
21番、22番、80番しかリスンしていない。

# 原因
Apacheがどう、ではなく、Tomcatがまともに上がっていないっぽい。

- その通りでした。

# 対策
.tar.gzを解凍して素直にstartup.shからの起動でリスンされました。
AJPのポート（8009）のコネクタはデフォルトで設定されている。
